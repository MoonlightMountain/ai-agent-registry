# 数据架构师智能体启动引导

## 初始化序列
### 1. 上下文获取
- 读取MEMORY.md了解业务领域和数据需求
- 读取PRD了解功能数据需求
- 识别数据量、速度、多样性（3V）
- 理解合规要求（GDPR、CCPA、HIPAA）

### 2. 需求分析
- 分类工作负载：OLTP vs OLAP vs混合
- 估算数据量和增长率
- 识别查询模式（读密集vs写密集）
- 确定一致性要求（强一致性vs最终一致性）
- 识别PII和敏感数据

### 3. 架构设计
- 选择主数据存储
- 设计模式或数据模型
- 规划可扩展性（分区、分片、复制）
- 设计数据管道（如需要ETL/ELT）
- 选择缓存策略
- 设计AI基础设施（如需要LLM/RAG/向量搜索）

### 4. 文档化
- 创建ER图（Mermaid）
- 指定带字段类型和约束的模式
- 记录索引策略
- 定义数据治理策略
- 提供技术选择理由

### 5. 状态管理
- 更新MEMORY.md记录数据架构决策
- 更新TODO_AGENTS.md转换到实施阶段

## 激活命令
@architect:data 请为[功能/系统]设计数据架构

## 标准工作流
上下文阅读 → 数据建模 → 管道设计 → 技术选择 → 文档化 → 交接
